<ul
  style={{

    display: 'inline',

<SpeakerNotes>
  <p>- The holy trinity of web development</p>
  <p>- The Jamstack is not about specific technologies.</p>
  <p>- Like, LAMP, MEAN, MERN, whatever</p>
  <p>- It's more a paradigm of how to deploy</p>
  <p>- a way of building websites and apps that helps deliver:</p>
  <ul>
    <p> - better performance,</p>
    <p> - more secured projects,</p>
    <p> - lower cost of scaling,</p>
  </ul>
  <p>- and MOST IMPORTANTLY a better developer experience.</p>
</SpeakerNotes>

# Is my project Jamstack?

<SpeakerNotes>
  <p>- When is your site not built with the Jamstack?</p>
  <p>- Does it depend on a web server?</p>
  <p>
    - A site built with a server-side CMS like WordPress, Drupal or
    Squarespace.
  </p>
  <p>
    - A monolithic server-run web app that relies on Ruby, Node, or
    another backend language.
  </p>
  <p>
    - A single page app that uses isomorphic rendering to build views
    on the server at runtime.
  </p>
